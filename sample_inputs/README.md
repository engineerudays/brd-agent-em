# Sample Inputs

This directory contains sample data and test inputs for the project.

## Structure

- `brds/` - Sample Business Requirements Documents
- `examples/` - Example input files for testing workflows

## Usage

1. Add sample BRD documents to `brds/` directory
2. Use these samples to test your n8n workflows
3. Include various formats and edge cases
4. Document the expected output for each sample

## File Formats

- PDF documents
- Word documents (.docx)
- Plain text files (.txt)
- Markdown files (.md)
- JSON structured inputs
